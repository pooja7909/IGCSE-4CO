About this hub

A complete, self-paced revision site for the Pearson Edexcel International GCSE in Computer Science (4CP0), first teaching September 2017, first examination June 2019. Aligned to the official specification so every topic, sub-topic and command word in your exam is covered.

🎯 What this qualification is for

The aims of the Pearson Edexcel International GCSE in Computer Science are to enable students to:

  • Understand and apply the fundamental principles and concepts of computer science — including abstraction, decomposition, logic, algorithms and data representation
  • Analyse problems in computational terms through practical experience of solving problems, designing, writing and debugging programs
  • Think creatively, innovatively, analytically, logically and critically
  • Understand the components that make up digital systems, and how they communicate with one another and with other systems
  • Understand the impacts of digital technology on the individual and on wider society
  • Apply mathematical skills relevant to computer science

Source: Pearson Edexcel 4CP0 Specification, Issue 1, November 2016.

📊 How the marks are distributed

Your final grade is built from three assessment objectives (AOs). Understanding their weights helps you spend your revision time wisely.

AOWhat it testsWeight
AO1 Demonstrate knowledge and understanding of the key principles of computer science
27.5%
AO2 Apply knowledge and understanding of key concepts and principles of computer science
42.5%
AO3 Analyse problems in computational terms — to make reasoned judgements; to design, program, test, evaluate and refine solutions
30%

Key takeaway: only ~27% of marks reward simple recall — over 70% test application and analysis. That's why this hub focuses heavily on quizzes, trace tables, error-spotting and the Code Lab rather than just notes.

How the AOs split across the two papers

  • Paper 1 — AO1: 21.5% · AO2: 21% · AO3: 7.5%
  • Paper 2 — AO1: 6% · AO2: 21.5% · AO3: 22.5%

So Paper 2 is overwhelmingly about applying and analysing, which is why writing real Python code is non-negotiable.

🗣 Know your command words

Examiners give marks for answering the question that was asked — not the question you wished was asked. Each Pearson command word has a precise meaning. Memorise these.

State / Give / NameOne short piece of information. No explanation needed.
IdentifySelect key information from a given stimulus.
DescribeGive an account — developed statements, no justification required.
ExplainJustify or exemplify a point. Marks need linked reasoning.
Compare / ContrastFind similarities and/or differences. Answer must address both items.
DiscussInvestigate an issue from multiple angles using reasoning.
AnalyseExamine elements in detail.
Assess / EvaluateWeigh factors then come to a supported judgement.
DeduceReach a conclusion from the information provided.

📝 The pseudocode you'll see in exams

Paper 1 uses Pearson's standardised pseudocode (Appendix 5 of the spec). It's language-neutral so the questions are fair to all students — Python, Java and C# users. The full reference will be available to you in Paper 2.

Quick reference — the syntax that always comes up

  • Variables: SET Counter TO 0
  • Arrays: SET ArrayValues TO [1, 2, 3, 4, 5] — indices start at 0
  • Input/Output: RECEIVE Name FROM (STRING) KEYBOARD · SEND 'Hello' TO DISPLAY
  • Selection: IF condition THEN ... ELSE ... END IF
  • Pre-conditioned loop: WHILE condition DO ... END WHILE
  • Post-conditioned loop: REPEAT ... UNTIL condition
  • Count loop: FOR i FROM 1 TO 10 DO ... END FOR
  • For-each loop: FOR EACH item FROM array DO ... END FOREACH
  • Function: FUNCTION name(params) BEGIN FUNCTION ... RETURN value END FUNCTION
  • Procedure: PROCEDURE name(params) BEGIN PROCEDURE ... END PROCEDURE
  • File I/O: READ MyFile.doc Record · WRITE MyFile.doc Answer1, Answer2
  • Operators: + - * / with MOD (modulo), DIV (integer division), ^ (exponent)
  • Relational: = <> > >= < <= (note: <> means "not equal to")
  • Logical: AND · OR · NOT

🔷 Flowchart symbols you need to know

Pearson uses these six symbols in all flowchart questions (Appendix 6 of the spec):

  • Oval / Pill — Start & End of an algorithm
  • Rectangle — A process to be carried out
  • Diamond — A decision to be made
  • Arrow — Shows the logical flow of the program
  • Rectangle with double sides — A subprocess defined elsewhere
  • Parallelogram — Input and/or output

🏆 How to actually score Grade 9

From years of mark-scheme patterns and the AO weightings above, Grade 9 in 4CP0 comes down to four habits:

  • Active recall, not re-reading. Closing the notes and writing them from memory teaches your brain ten times faster than re-reading. Use the quizzes and trace tables.
  • Trace before you code. Walk through every algorithm by hand first. Examiners reward visible working far more than a final answer.
  • Master the vocabulary. Marks come from precise terminology — "fetch–decode–execute cycle", "lossless compression", "two's complement". Vague phrasing loses marks.
  • Past papers, timed. Once you've done the topic work, sit a past paper under timed conditions, then mark it strictly. Repeat at least 6 papers before exam day.

  • Topic 1: Problem solving — algorithms, decomposition, abstraction
